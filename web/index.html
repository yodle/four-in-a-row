What Is Four in a Row?

Four in a Row is a simple game in which players take alternating turns dropping a single play token into one of seven columns in a vertically oriented board. As the name suggests, the first player to align four adjacent play tokens in a horizontal, vertical, or diagonal fashion wins the game.

While the rules are simple, coming up with an efficient AI to play the game well is a challenging (and fun) endeavor. If you're up to the challenge, try to take on one of our engineer-submitted AIs in a one-on-one deathmatch.

How to Play

There are three ways to play:

  • Write a javascript AI in our online code editor
  • Use one of the Python, or Ruby or Go clients
  • Roll your own in your language of choice against the Four in a Row Rest API

REST API Documentation

The REST API consists of two primary API methods and one supporting method for testing purposes. Each method returns the same information in the response, which is encoded in JSON format. Where resource identifiers are required in the URL, they are denoted as :param_description. All API URLs are relative to http://challenge.yodle.com:8080

Standard JSON Response

Value
ROWS
The number of rows in the game board, always 6

The number of columns in the game board, always 7

The unique game identifier, used in the other API methods. Note: this value is a string type.

Your player number (1 or 2), corresponding to the turn order
gameOver
0 if the game is not over, otherwise the winner's player number
nickname
The nickname you supplied to the init API, used for the leaderboard
turn
Whose turn it is - will always be your player number except if the opponent AI is thinking during a request to get the game state
moves
How many moves have been played in the current game
lastMove
JSON object describing the last move made. Fields are:
  • row - row of last move
  • col - column of last move
  • player - player number that made last move
  • moves - total number of moves made
moveList
An array containing the move history in ascending chronological order
ai
The opponent AI level
board
The current board state; a two-dimensional array with columns (left-to-right) on the first dimension and rows (top of the board at lowest index) on the second
message
A message describing the end of the game, or undefined if the game is not over.
error
A description of the error that has occurred, or undefined if there is no error. Errors (like invalid moves) cause you to lose.

API Method: POST /game/init/:aiLevel

Initiates a new game. AI level should be an integer between 1 and 6, with 1 being the easiest and 6 being the hardest. A nickname may be supplied but is not required. The response will contain the challenger player's player number, denoting whether the challenger player is the first or second to make a move. If the computer player is the first player, the board will reflect the computer player's first move in the response.

Example init request json: 
{
    nickname: "name or email"  
}

API Method: POST /game/move/:gameId

Inserts a single play token into the board, into the column designated by the move parameter (between 0 and 6). The gameId segment of the uri should correspond to a unique game identifier returned by a call to the init/ API. Once the move has been made, the server will solicit a move from the computer opponent before returning. Thus, after a call to the init/ or move/ APIs, it will always either be the challenger player's turn or the game will be over.

Example move request json: 
{
    move: 3 
}

API Method: GET /game/state/:gameId

Retrieves the current state of the game corresponding to the gameId parameter.
