Welcome to the Yodle Four in a Row Challenge

What Is It?

Think you've got what it takes to match our engineers in a battle of wits? Take on the Yodle Four in a Row Challenge!

Four in a Row is a simple game in which players take alternating turns dropping a single play token into one of seven columns in a vertically oriented board. As the name suggests, the first player to align four adjacent play tokens in a horizontal, vertical, or diagonal fashion wins the game.

While the rules are simple, coming up with an efficient AI to play the game well is a challenging (and fun) endeavor. If you're up to the challenge, try to take on one of our engineer-submitted AIs in a one-on-one deathmatch.

How to Play

There are three ways to play:

  • Write a javascript AI in our online code editor
  • Use one of the Java, Python, or Ruby clients
  • Roll your own in your language of choice against the Four in a Row Rest API

REST API Documentation

The REST API consists of two primary API methods and one supporting method for testing purposes. Each method returns the same information in the response, which is encoded in JSON format. Where resource identifiers are required in the URL, they are denoted as :param_description. POST parameters are denoted in (parentheses) after the URL. All API URLs are relative to http://TODO_DOMAIN_HERE.com.

movesHow many moves have been played in the current game
lastMoveThe most recent move (denoted by column), or null on the first turn
moveListAn array containing the move history in ascending chronological order
aiThe opponent AI level
boardThe current board state; a two-dimensional array with columns (left-to-right) on the first dimension and rows (top of the board at lowest index) on the second
errorA description of the error that has occurred. This is only defined when there is an error. Errors (like invalid moves) cause you to lose.

API Method: POST /game/init/:aiLevel (nickname)

Initiates a new game. AI level should be an integer between 1 and 6, with 1 being the easiest and 6 being the hardest. A nickname may be supplied for the leaderboard (strongly suggested). The response will contain the human player's player number, denoting whether the human player is the first or second to make a move. If the computer player is the first player, the board will reflect the computer player's first move in the response.

API Method: POST /game/move/:gameId (move)

Inserts a single play token into the board, into the column designated by the move parameter (between 1 and 7). The gameId segment of the uri should correspond to a unique game identifier returned by a call to the init/ API. Once the move has been made, the server will solicit a move from the computer opponent before returning. Thus, after a call to the init/ or move/ APIs, it will always either be the human player's turn or the game will be over.

API Method: GET /game/state/:gameId

Retrieves the current state of the game corresponding to the gameId parameter.
